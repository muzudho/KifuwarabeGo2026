# リリース手順

このソリューションから利用者へ配布するZIPは、GUI版とEngine版の2種類です。

```text
KifuwarabeGo2026.Gui-v<version>-win-x64.zip
KifuwarabeGo2026.Engine-v<version>-win-x64.zip
```

- GUIだけ欲しい人にはGUI版を配布します。
- 思考エンジンだけ欲しい人にはEngine版を配布します。
- 両方欲しい人には2つとも案内します。
- `KifuwarabeGo2026.Shared` と `KifuwarabeGo2026.Gui.Communication.Cgos` は単独では配布しません。

## 配布物の構成

GUIをpublishすると、`KifuwarabeGo2026.Shared.dll` が自動的に入り、CGOS通信コンポーネントも `Tools\Cgos` へ自動的にpublishされます。

```text
GUI版publish/
├─ KifuwarabeGo2026.Gui.exe
├─ KifuwarabeGo2026.Shared.dll
├─ Content/
└─ Tools/
   └─ Cgos/
      ├─ KifuwarabeGo2026.Gui.Communication.Cgos.exe
      ├─ KifuwarabeGo2026.Gui.Communication.Cgos.dll
      ├─ KifuwarabeGo2026.Gui.Communication.Cgos.deps.json
      └─ KifuwarabeGo2026.Gui.Communication.Cgos.runtimeconfig.json
```

Engineをpublishすると、`KifuwarabeGo2026.Shared.dll` が自動的に入ります。

```text
Engine版publish/
├─ KifuwarabeGo2026.Engine.exe
├─ KifuwarabeGo2026.Engine.dll
├─ KifuwarabeGo2026.Shared.dll
├─ KifuwarabeGo2026.Engine.deps.json
└─ KifuwarabeGo2026.Engine.runtimeconfig.json
```

ここに挙げたものだけを抜き出さず、それぞれのpublishフォルダーを丸ごとZIPにしてください。依存DLLやネイティブライブラリーも必要です。

## 1. リリース前の確認

リポジトリのルートで作業します。

```powershell
git status --short
dotnet build KifuwarabeGo2026.slnx
```

意図しない変更がないことと、ビルドが警告0・エラー0で終わることを確認します。

バージョンを上げる場合は、次のプロジェクトファイルにある `Version`、`AssemblyVersion`、`FileVersion` を確認します。

- `KifuwarabeGo2026.Gui/KifuwarabeGo2026.Gui.csproj`
- `KifuwarabeGo2026.Engine/KifuwarabeGo2026.Engine.csproj`
- `KifuwarabeGo2026.Shared/KifuwarabeGo2026.Shared.csproj`
- `KifuwarabeGo2026.Gui.Communication.Cgos/KifuwarabeGo2026.Gui.Communication.Cgos.csproj`

## 2. GUI版をpublishする

```powershell
dotnet publish KifuwarabeGo2026.Gui\KifuwarabeGo2026.Gui.csproj -c Release -r win-x64 --self-contained false
```

出力先:

```text
KifuwarabeGo2026.Gui\bin\Release\net8.0-windows\win-x64\publish
```

このコマンドの中でCGOS通信コンポーネントも自動的にpublishされます。CGOSプロジェクトに対する別の `dotnet publish` は不要です。

## 3. Engine版をpublishする

```powershell
dotnet publish KifuwarabeGo2026.Engine\KifuwarabeGo2026.Engine.csproj -c Release -r win-x64 --self-contained false
```

出力先:

```text
KifuwarabeGo2026.Engine\bin\Release\net8.0\win-x64\publish
```

`KifuwarabeGo2026.Shared` に対する別の `dotnet publish` は不要です。

## 4. publish成果物を確認する

GUI版の必須ファイルを確認します。

```powershell
$guiPublish = 'KifuwarabeGo2026.Gui\bin\Release\net8.0-windows\win-x64\publish'
Get-Item -LiteralPath `
    "$guiPublish\KifuwarabeGo2026.Gui.exe", `
    "$guiPublish\KifuwarabeGo2026.Shared.dll", `
    "$guiPublish\Tools\Cgos\KifuwarabeGo2026.Gui.Communication.Cgos.exe", `
    "$guiPublish\Tools\Cgos\KifuwarabeGo2026.Gui.Communication.Cgos.dll", `
    "$guiPublish\Tools\Cgos\KifuwarabeGo2026.Gui.Communication.Cgos.deps.json", `
    "$guiPublish\Tools\Cgos\KifuwarabeGo2026.Gui.Communication.Cgos.runtimeconfig.json"
```

同梱されたCGOS通信コンポーネントを確認します。

```powershell
& "$guiPublish\Tools\Cgos\KifuwarabeGo2026.Gui.Communication.Cgos.exe" --help
```

Engine版の必須ファイルを確認します。

```powershell
$enginePublish = 'KifuwarabeGo2026.Engine\bin\Release\net8.0\win-x64\publish'
Get-Item -LiteralPath `
    "$enginePublish\KifuwarabeGo2026.Engine.exe", `
    "$enginePublish\KifuwarabeGo2026.Shared.dll"
```

可能ならpublishフォルダー外へコピーした状態でも、GUIの起動、CGOS接続画面、EngineのGTP応答をスモークテストします。

```powershell
@('protocol_version','name','version','boardsize 9','clear_board','quit') |
    & "$enginePublish\KifuwarabeGo2026.Engine.exe"
```

`--self-contained false` で作っているため、利用者のWindowsには対応する.NETランタイムが必要です。

## 5. ZIPを作る

publishフォルダー自体ではなく、その中身がZIP直下へ展開される形にします。

```powershell
$version = '1.1.0'
Compress-Archive -Path "$guiPublish\*" -DestinationPath "KifuwarabeGo2026.Gui-v$version-win-x64.zip"
Compress-Archive -Path "$enginePublish\*" -DestinationPath "KifuwarabeGo2026.Engine-v$version-win-x64.zip"
```

同名ZIPがすでにある場合、上書き前に中身と退避先を確認してください。

## 6. リリースへ登録する

リリースページで同じバージョンのタグとリリースを作成し、次の2ファイルを添付します。

- `KifuwarabeGo2026.Gui-v<version>-win-x64.zip`
- `KifuwarabeGo2026.Engine-v<version>-win-x64.zip`

説明には、GUI版とEngine版が別配布であること、両方使う場合は2つとも必要であること、必要な.NETランタイムのバージョンを記載します。

## 失敗したとき

- GUI版に `Tools\Cgos` がない場合は、`KifuwarabeGo2026.Gui.csproj` の `PublishCgosCommunication` ターゲットを確認します。
- GUIからCGOSを起動できない場合は、`Tools\Cgos` の4つの必須ファイルと、GUIのログに出た不足ファイル名を確認します。
- `Shared.dll` がない場合は、GUIまたはEngineからSharedへの `ProjectReference` とpublishログを確認します。
- ファイルがロックされた場合は、GUI、Engine、CGOS通信クライアント、Visual Studioのデバッグ実行を終了してからpublishをやり直します。
