# 基本方針

最終更新: 2026-07-07

## プロジェクト目的

`KifuwarabeGo2026` は、コンピューター囲碁の思考エンジンとして作る。

## 技術方針

- フレームワークには MonoGame を使う。
- MonoGame を使う理由は、ビデオゲームと同じ作り方で画面を表示できるため。
- GTP 実装は、以前の Go 言語版を参考にしつつ、C# で一から書き直す。

## GTP 方針

コンピューター囲碁では、対局プロトコルとして Go Text Protocol (GTP) を使う。

- 標準入力から GTP コマンドを受け取る。
- 標準出力へ GTP 応答を返す。
- 標準エラー出力は使わない。
- 標準出力へデバッグログを出さない。

初期実装で扱う候補の GTP コマンド:

- `protocol_version`
- `name`
- `version`
- `known_command`
- `list_commands`
- `boardsize`
- `clear_board`
- `komi`
- `play`
- `genmove`
- `quit`

## アプリケーション操作方針

標準入力と標準出力は GTP 専用にするため、アプリケーション操作には使わない。

画面上の操作には以下を使う。

- マウス
- ボタン
- テキストボックス
- 必要に応じた画面内ログ表示

## 分離したい責務

- GTP 入出力層: 標準入力と標準出力だけを扱う。
- GTP コマンドパーサー: コマンド名と引数を解析する。
- GTP コマンド実行層: 解析済みコマンドを囲碁エンジンへ渡し、応答を作る。
- 囲碁ロジック層: 盤、石、合法手、着手、生成手を扱う。
- MonoGame UI 層: 画面表示とマウス・ボタン・テキストボックス操作を扱う。
