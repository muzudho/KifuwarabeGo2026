# 続きはここから

最終更新: 2026-07-09

## 現在の前提

- このリポジトリーでは、コンピューター囲碁の思考エンジン `KifuwarabeGo2026` を作る。
- 実装言語は C#。
- 画面表示には MonoGame フレームワークを使う。
- 対局プロトコルには Go Text Protocol (GTP) を使う。
- GTP の入出力は標準入力と標準出力を使う。
- 標準エラー出力は使わない。
- アプリケーション操作には標準入出力を使わず、MonoGame の画面上 UI を使う。

## 決まったこと

- 標準入力と標準出力は GTP 専用にする。
- デバッグログや画面操作結果を標準出力へ混ぜない。
- 画面操作は、マウス、ボタン、テキストボックスなどで行う。
- 以前の Go 言語版 GTP 実装を参考にしつつ、今回は C# で一から書き直す。
- GUI と思考エンジンは別プロセスに分ける方針。
- GUI から思考エンジン `.exe` を起動し、GTP コマンドと応答で連携する。
- 最初の思考エンジンは、強さより別プロセス連携確認を優先し、ランダム合法手エンジンで始める。

## 直近の完了

- GUI のセットアップ画面で、黒番・白番を `Human` / `Computer` から選べるようにした。
- 既定の対局者設定は、黒番 `Human`、白番 `Computer`。
- `KifuwarabeGo2026.Engine` コンソールプロジェクトを追加した。
- エンジンは `protocol_version`、`name`、`version`、`boardsize`、`clear_board`、`play`、`genmove`、`quit` に対応した。
- GUI プロジェクト内に `GtpEngineClient`、`GtpEngineSettings`、`GtpResponse`、`GtpCoordinate` を追加した。
- エンジン単体で `boardsize 9`、`play black D4`、`genmove white` の GTP 応答を確認した。

## 現在の未接続部分

- GUI の対局ループから `GtpEngineClient` はまだ呼び出していない。
- 対局開始時の `boardsize` / `clear_board` 送信はまだ未実装。
- 人間の着手後に `play black D4` のような GTP コマンドを送る処理はまだ未実装。
- コンピューター手番で `genmove white` を送り、返ってきた手を盤へ反映する処理はまだ未実装。
- `GtpEngineClient` はログ保存先として実行ファイル基準の `logs/gtp.log` を使う実装だが、GUI からの実使用確認はまだ。
