# 【むずでょ個人用】リリース方法


## バージョン上げ

ソリューション・エクスプローラーからプロジェクト名を右クリックして「プロパティ」を開き、  
［パッケージ　＞　全般］を選んで、下の方に、  
［アセンブリバージョン］と［ファイルバージョン］を入力する欄があるぜ。  


## 実行ファイル、ZIPファイル作成

```powershell
dotnet publish KifuwarabeGo2026\KifuwarabeGo2026.csproj -c Release -r win-x64 --self-contained false

dotnet publish KifuwarabeGo2026.Engine\KifuwarabeGo2026.Engine.csproj -c Release -r win-x64 --self-contained false

// 注意点として、GUIから Engine を起動する設定は `Content\GtpEngines\gtp-engine-list.json` に入っています。
// publish 後に配布するなら、GUI の設定画面で Engine の publish 済み exe を指定するか、
// 配布フォルダ内で相対パスが合うように Engine を置く必要があります。
```

👆　出力先はだいたいここです。  
publish という名前のフォルダーが作られて、その中に実行ファイルが入ってます。  
※ソリューション・エクスプローラーからは見れないことがある。  

publish フォルダーを、  
E:\github.com\muzudho\YukaiLarkStateTransitionDiagram\Release  
へ、カット＆ペースト（＾～＾）  

リリースする ZIP ファイル名へのリネーム例：  

```
KifuwarabeGo2026.Gui-v1.0.0-win-x64
```


## Git Hub へリリース

https://github.com/muzudho/YukaiLarkStateTransitionDiagram/releases  
みたいなページへ行って、 `Draft a new release` ボタンを押す。  

`Tag: Select tag` ボタンを押す。  
`Create new tag` ボタンを押す。  
バージョン番号を入れる。 `v1.0.0` みたいな感じで。  

次は、`Release title` に、リリースのタイトルを入れる。  

`Describe this release` に、リリースの説明を入れる。  

.zip ファイルをドラッグ＆ドロップでアップロードする。
